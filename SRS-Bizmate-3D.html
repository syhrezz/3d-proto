Pendahuluan

Bizmate 3D adalah solusi inovatif untuk industri retail furnitur yang memungkinkan pelanggan untuk memvisualisasikan produk dalam ruang nyata menggunakan teknologi Augmented Reality (AR) dan 3D rendering.

Tujuan utama dari dokumen ini adalah untuk menyelaraskan visi antara pengembang, desainer, dan stakeholder dalam membangun sistem yang skalabel dan user-friendly.

Tech Stack

Frontend Framework

React.js 18 - Digunakan untuk membangun antarmuka pengguna yang reaktif dan komponen-komponen UI yang modular.

3D & AR Engine

Google Model Viewer - Standar industri untuk visualisasi 3D web yang menjamin konsistensi skala antara Android (WebXR) dan iOS (Quick Look).

Styling

Tailwind CSS - Framework CSS utility-first untuk desain yang konsisten, modern, dan responsif.

Navigation

React Router DOM - Untuk manajemen navigasi antar halaman (Catalog, Detail, AR View).

Aktor & Peran

Administrator

Bertanggung jawab penuh atas manajemen konten dan operasional sistem di belakang layar.

  • Mengelola katalog produk (tambah, ubah, hapus).
  • Memantau statistik penggunaan fitur AR.
  • Mengelola akses untuk tim 3D Designer.

Customer (Pelanggan)

Pengguna akhir yang berinteraksi langsung dengan katalog produk dan fitur WebXR.

  • Menelusuri produk furnitur dalam katalog.
  • Melihat detail produk dalam mode 3D Interaktif.
  • Menempatkan furnitur secara virtual di ruangan menggunakan AR.

3D Designer (Operational Role)

Personel kunci dalam menjaga kualitas aset visual 3D.

  • Optimasi poligon model 3D agar ringan di load browser.
  • Konversi format file ke .GLB (Android/Web) dan .USDZ (iOS).
  • Setting material (PBR) agar terlihat realistis di bawah pencahayaan AR.

Kriteria Penerimaan (AC)

AC - Administrator

Admin dapat mengunggah file 3D dengan validasi otomatis ukuran file (maks 10MB).

Dashboard admin menampilkan preview 3D sebelum produk dipublikasikan ke katalog.

AC - Customer

Customer dapat merotasi dan melakukan zoom pada model 3D dengan responsivitas tinggi (60 FPS).

Fitur "Lihat di Ruangan" hanya muncul jika perangkat mendukung WebXR atau AR Core/AR Kit.

Alur Operasional & Unggah Produk

1. Persiapan Aset 3D

Membutuhkan: 3D Designer.
Designer membuat model furnitur dengan detail presisi skala 1:1. Semua tekstur harus dipanggang (baked) untuk performa optimal.

2. Optimasi & Ekspor

Model diekspor dalam dua format utama oleh tim desainer:

  • Format Utama (Web/Android): .GLB
  • Format Apple (iOS): .USDZ
  • Target Ukuran: 2MB - 5MB (Maksimum 10MB)

3. Unggah ke CMS oleh Admin

Admin menginput data produk (nama, deskripsi, harga) dan mengunggah file yang sudah disiapkan desainer. Sistem akan menjalankan pengecekan kompatibilitas file.

4. Publikasi & Sinkronisasi

Setelah divalidasi, produk akan muncul secara real-time di katalog pelanggan dan fitur AR siap digunakan.

Solusi Kompatibilitas iOS

Mengapa WebXR Tidak Berjalan di iOS?

Apple Safari saat ini belum mengaktifkan API WebXR Device secara default. Untuk memberikan pengalaman AR yang lancar di iPhone dan iPad, Bizmate 3D menggunakan strategi Hybrid AR:

Android & Desktop

Menggunakan WebXR API untuk penempatan furnitur secara presisi dengan fitur hit-test.

Apple iOS

Menggunakan AR Quick Look melalui format file .USDZ untuk integrasi sistem native Apple.
